Autonomous Robotic Arm

4-Degrees of Freedom manipulated using Microsoft Kinect

The aim of this project was to automate redundant tasks using a robotic arm. The equipment we were provided with were a Microsoft Kinect camera and a dynamixel arm with four degrees of freedom - RRR:R. Our task was to design an efficiemt gripping mechanism for the robot and detect different colored cubes using depth maps and images streamed from the overhead Kinect. Once done, we were to design a Spline -based trajectory algorithm for smooth motion of the arm and complete 5 assigned tasks whose details were predefined.

Object Detection

Depth Maps and Camera data was gathered from the Kinect sensor. OpenCV - Python was used for the block detection based on different colors. Once the detection part was implemented, we started tuning the HSL parameters for each of the given colors. We soon realized that, under different lighting conditions, the same thresholding values for each color doesn't apply well. To overcome this, a neural network with one hidden layer (with 10 neurons) was implemented to be able to fine tune the HSL values for each of the colors. Once trained, there was a substantial increase in block detection accuracy(~43%), hence giving us near perfect color detection in most lighting conditions.

Gripper Design | Trajectory Planning

The Gripper Design was a crucial component in this project. The arm was required to maneuver dexterously in a 300 x 300 x 300 cm cube. With careful FEM modelling of the gripper, we managed a 24% weight reduction on the gripper while maintaining it's rigidity. The CAD model of the gripper is attached below.
Spline based trajectory planning was implemented for smooth motion of the arm. A teach-and-repeat paradigm was implemented on the arm which had a performance accuracy of nearly 87%. Cubic spline generation was used for moving between keypoints in the trajectory. For various tasks, the forward and inverse kinematics of the arm were calculated and a spline trajectory was generated between the current position and the destination. LCM was used to send commands to the motors and Finite State Machines were implemented for each individual tasks. A Python based GUI was used to manually input which task needed to be performed.

Pendulum Control Simulation

Simulation/ Control of a single and a double pendulum

One of the most trivial control problems is that of a pendulum. In this project, a simple PID control structure was written for controlling the pendulum. Some of the prevalent integrators were written for the physics of the simulation : Euler Integrator, Verlet Integrator, Velocity Verlet Integrator and the Runge Kutta (RK4) integrator. Each of the following can be selected by choosing keys 0-4 while running the demo for a single pendulum. Only the RK4 integrator has been written for the Double Pendulum simulation. For a given simulation, it can be seen that the RK4 integrator gives the most accurate representation of the system, whereas the Euler integrator is unstable under all conditions. In the case of the single pendulum, PID tuning for only the velocity verlet integrator was done.

Simulation/ Control of the Cart-Pole Mechanism

The cart-pole system is a classical case of the control of the inverted pendulum mechanism. This project is similar in nature to Pendulum Control. The velocity verlet integrator is used for the physics of this simulation. The PID takes some time to stabilize the cart once the integrator is switched on, however, once its stable, it is difficult to cause the pole to topple from the cart.
